Increment 1

Minimal Server Scaffold

Build the smallest possible backend application that runs locally, responds to a health check, and defines a placeholder extraction endpoint.

Goal

Build a minimal TypeScript Express server with:

  • GET /health → returns { ok: true }
  • POST /extract → returns 501 not implemented

Why this step exists

Before adding OCR, LLM calls, traces, or UI, you need to understand how a backend app starts, accepts requests, and returns responses. This is the foundation for everything else.

What we are building

A very small backend service with no business logic yet.

Included

  • Local server
  • One health endpoint
  • One placeholder extract endpoint

Not included yet

  • OCR
  • File upload
  • LLM calls
  • Frontend/UI

Files introduced

src/
  app.ts
  server.ts
  routes/
    extract.ts

evals/
  manual_checks.md

Packages

npm install express
npm install -D typescript tsx @types/node @types/express

Codex prompt

Create the smallest possible TypeScript Express server.

Requirements:
- Add src/app.ts, src/server.ts, src/routes/extract.ts
- GET /health returns { ok: true }
- POST /extract returns 501 with { "message": "not implemented" }
- Add package.json script: "dev": "tsx src/server.ts"
- Add minimal tsconfig.json
- Keep code simple and readable
- Do not add OCR, uploads, frontend, or extra features

How to run

npm run dev

Test in browser

http://localhost:3000/health

Expected response:

{ "ok": true }

Test in terminal

curl -X POST http://localhost:3000/extract

Expected response:

{ "message": "not implemented" }

Manual eval checks

  • GET /health returns { ok: true }
  • POST /extract returns 501 not implemented

Ship it criteria

  • Server starts without errors
  • /health works in the browser
  • /extract returns the correct placeholder response
  • Manual eval checks pass
  • Code is committed to Git

Do not add yet

  • OCR
  • File uploads
  • LLM calls
  • Tracing
  • Eval automation
  • Frontend/UI
  • Multiple routes or abstractions

Concepts to understand before moving on

  1. What file starts the server?
  2. What is a route?
  3. What does npm run dev actually execute?
  4. Why include /extract even though it does nothing yet?
  5. Why is this step intentionally minimal?

Workflow recap

Write notes → Define eval → Generate code → Run → Verify → Commit
